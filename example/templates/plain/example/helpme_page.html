Patch verification

Archive verification:

  • Allowed extensions are: 
    .zip
    , 
    .tar.gz
    , 
    .tar.bz2
    and 
    .tar.xz
  • Must contain a patch file called 
    unified_diff.patch
  • All files in the archive must be placed at the top level (i.e., not in folders).

Allowed extra content:

  • Additional files with the extensions: 
    .qml
    , 
    .js
    , 
    .png
    , 
    .svg
     and 
    .qm
    • 
    
  • Patch translation 
    qm
    -files should be named 
    translation_LANG.qm
     or 
    PATCHNAME-LANG.qm
    .
    • 
    
  • An optional icon for a Patch should be named 
    main.png
     or 
    main.svg
    , additionally a 
    main-light.png
     or 
    main-light.svg
     icon may be included for "light" (i.e., bright) ambiences.
    • 
    
  • An optional QML-file, which allows the user to configure the settings of a Patch should be named 
    main.qml
    .
    
    Additional QML-pages may be called from 
    main.qml
    . All extra files are located in same directory.
    • 
  


Translation tips and tricks:

  
    
    
  • Add: 
    import org.SfietKonstantin.patchmanager 2.0
    • 
    
  • In the root item add: 
    property bool pmTranslationLoaded: PatchManagerTranslator ? PatchManagerTranslator.installTranslator("my-patch-name") : false
    • 
    
  • In the first visible text item replace its text with: 
    pmTranslationLoaded ? qsTr("Translated text") : "Please update Patchmanager!"
    • 
    
  • If the translation is not loaded, try using 
    qsTranslate()
     strings instead: 
    pmTranslationLoaded ? qsTranslate("pm", "Translated text") : "Please update Patchmanager!"
    • 
  


Project fields verification:

  
    
    
  • The "Name" field must solely comprise alphanumeric characters, the underscore and the hyphen.
    
    I.e., available chars are 
    a-z
    , 
    A-Z
    , 
    0-9
    , 
    _
     and 
    -
    .
    
    The name value is only used internally.  For a pretty name use the "Display name" field.
    • 
    
  • The version must comprise three natural numbers (including 0) separated by two dots, for example 
    1.2.3
    , 
    0.0.2
     or 
    47.11.123
    .
    • 
  


Additional information and guidance is avalilable at GitHub.
